# Step1: 自己紹介を作ろう

## このステップの目的

Gitの基本的なワークフローを一通り体験します。

```
branch → add → commit → push → Pull Request → merge
```

この流れはチーム開発で最も頻繁に使うパターンです。「自己紹介ファイルを追加する」というシンプルな作業を通じて、一連の流れを身につけましょう。

## 前提条件

- このリポジトリが `git clone` 済みであること
- `main` ブランチにいること（`git branch` で確認できます）

---

## 手順

### 1. ブランチを作成する

```bash
git checkout -b 自分の名前
```

> **解説**: `git checkout -b` は新しいブランチを作成して、そのブランチに切り替えるコマンドです。`main` ブランチを直接変更せず、自分専用の作業スペースを作ります。

例:
```bash
git checkout -b tanaka
```

### 2. 自己紹介ファイルを作成する

`自己紹介/sample.md` をコピーして、自分の名前のファイルを作成します。

```bash
cp 自己紹介/sample.md 自己紹介/自分の名前.md
```

例:
```bash
cp 自己紹介/sample.md 自己紹介/tanaka.md
```

### 3. 内容を編集する

コピーしたファイルを好きなエディタで開いて、自分の情報を記入しましょう。

### 4. 変更をステージングする

```bash
git add 自己紹介/自分の名前.md
```

> **解説**: `git add` は変更したファイルを「ステージングエリア」に追加するコマンドです。コミットに含めたいファイルを選択する作業です。

### 5. コミットする

```bash
git commit -m "自己紹介を追加"
```

> **解説**: `git commit` はステージングした変更を記録するコマンドです。`-m` でコミットメッセージを指定します。どんな変更をしたか分かるメッセージを書きましょう。

### 6. リモートにプッシュする

```bash
git push origin 自分の名前
```

> **解説**: `git push` はローカルのコミットをリモートリポジトリ（GitHub）にアップロードするコマンドです。`origin` はリモートリポジトリの名前、その後にブランチ名を指定します。

### 7. Pull Request を作成する

1. GitHubのリポジトリページを開く
2. 「Compare & pull request」ボタンをクリック
3. タイトルと説明を記入する（例: 「田中の自己紹介を追加」）
4. 「Create pull request」ボタンをクリック

> **解説**: Pull Request（PR）は、自分の変更を `main` ブランチに取り込んでもらうためのリクエストです。チームメンバーにレビューしてもらい、問題がなければマージします。

### 8. レビュー後にマージする

レビュアーから承認（Approve）をもらったら、GitHub上で「Merge pull request」ボタンをクリックしてマージします。

---

## まとめ

| ステップ | コマンド | やっていること |
|---|---|---|
| ブランチ作成 | `git checkout -b 名前` | 作業スペースを作る |
| ステージング | `git add ファイル` | コミットするファイルを選ぶ |
| コミット | `git commit -m "メッセージ"` | 変更を記録する |
| プッシュ | `git push origin ブランチ名` | GitHubにアップロードする |
| PR作成 | GitHub上で操作 | レビューを依頼する |
| マージ | GitHub上で操作 | mainに統合する |

この流れがGitを使ったチーム開発の基本です。次のステップに進みましょう！
